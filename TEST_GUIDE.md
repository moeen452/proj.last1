# 📝 خطوات الاختبار
############################################

### الخطوة 1: اختبر السيرفر
- قم بتشغيل: `GET http://localhost:3000/`
- يجب أن ترى: `"message": "API is running ✅"`

### الخطوة 2: سجّل حساب جديد
- قم بتشغيل: `POST /auth/register`
- ستظهر في Terminal رسالة:
  ```
  🔗 Verify: http://localhost:3000/api/v1/auth/verify-email?token=xxxxxxxxxx
  ```
- **انسخ هذا الرمز كاملاً** (الجزء بعد `token=`)

### الخطوة 3: تحقق من الإيميل
- استبدل `PAST_TOKEN_FROM_TERMINAL_HERE` برمز من Terminal
- قم بتشغيل: `GET /auth/verify-email?token=...`
- يجب أن ترى: `"message": "تم تأكيد الإيميل ✅"`

### الخطوة 4: ادخل الحساب
- قم بتشغيل: `POST /auth/login`
- ستحصل على `accessToken` طويل جداً

### الخطوة 5: انسخ التوكن
- اذهب للأعلى في ملف test.http
- استبدل جميع:
  ```
  Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
  ```
  بالتوكن الجديد من الخطوة 4

### الخطوة 6: اختبر باقي الطلبات
- `GET /auth/me` — يجب تشوف بيانات المستخدم
- `POST /startups` — أنشئ شركة ناشئة
- `GET /startups/my` — شوف بيانات الشركة

# 🔑 النقاط المهمة
############################################

✅ **الترتيب:**
1️⃣ Register → 2️⃣ Verify → 3️⃣ Login → 4️⃣ Copy Token → 5️⃣ Test Protected Routes

✅ **رابط Terminal:**
- ظهور الرمز في Terminal **مهم جداً**
- إذا لم تشوف الرمز، تأكد من:
  - السيرفر يعمل (npm run dev)
  - لا فيه errors في Terminal

✅ **التوكن:**
- صحتين انتبه للمسافات قبل وبعد التوكن
- لا تنسى كلمة "Bearer " قبل التوكن
- التوكن ينتهي كل 15 دقيقة

# 💡 نصائح الاستخدام
############################################

**إذا حصلت على خطأ 400:**
```json
{
  "success": false,
  "error": {
    "message": "الرمز غير صحيح أو انتهت صلاحيته",
    "code": "INVALID_TOKEN"
  }
}
```
→ استخدم رمز جديد من Terminal

**إذا حصلت على خطأ 401:**
```json
{
  "error": { "message": "يجب تسجيل الدخول أولاً" }
}
```
→ التوكن ناقص أو خاطئ

**إذا حصلت على 409:**
```json
{
  "error": { "message": "هذا الإيميل مسجل مسبقاً" }
}
```
→ استخدم إيميل جديد
